Setting up your View

Anatomy of a Basic View

Views in the framework differ very little to a standard webpage, just a few extras here and there and you have a view which is compatible with the framework.

The framework is designed to load each page dynamtically copying elements of the html from each page into the DOM, this allows for seamless transitions and retention of JavaScript memory and application state.

Lets start with the basic layout of a view, static headers and footers can feature in the application but this is discussed later in the Working with Headers and Footers section. 


<!DOCTYPE html>
<html lang="en">
	<head>
		<meta charset="utf-8">
1		<meta name="viewport" content="width=device-width, initial-scale=1.0, maximum-scale=1.0">
		<meta name="apple-mobile-web-app-capable" content="yes" />
		
2		<title>Mando Mobile | View Name</title>
		
3		<link rel="icon" href="/Images/favicon.ico" type="image/x-icon" />
		<link rel="apple-touch-icon" href="/Images/ios-icon-57x57.png" />
		<link rel="apple-touch-icon" sizes="72x72" href="/Images/ios-icon-72x72.png" />
		<link rel="apple-touch-icon" sizes="114x114" href="/Images/ios-icon-144x144.png" />
		
4		<link type="text/css" rel="stylesheet" href="/styles/mandoMobile.css" />
		<link type="text/css" rel="stylesheet" href="/styles/screen.css" />
		
		<!-- jQuery -->
5		<script type="text/javascript" src="/scripts/jquery-1.8.2.min.js"></script>
		<script type="text/javascript" src="/scripts/jquery.ba-hashchange.js"></script>

		<!-- Modernizr -->
6		<script type="text/javascript" src="/scripts/modernizr.js"></script>

		<!-- MandoMobile Platform -->
7		<script type="text/javascript" src="/scripts/mm/mandoMobile.js"></script>
		<script type="text/javascript" src="/scripts/mm/mandoMobile-config.js"></script>

	</head>

	<body>
		
8		<div data-view="view_id">
			
9			<div data-view-inner>
				
			</div>
			
		</div>
		
	</body>
</html>
  • 1 Meta tags: These Meta tags are required to ensure scale remains constant on mobile devices, this will have no effect on desktop representations.
  • 2 Title: Nothing special here really, but note that this title is used by the framework to change the window's title as it asyncronously downloads each page in your application.
  • 3 Apple / Favicons: Not essential but you should supply some icons for the browser and iOS homescreen.
  • 4 MandoMobile framework Stylesheet: The Absolute necessary styles are applied via JavaScript in the application, but it is strongly adviced you include this very small StyleSheet in your application as it provides some default structure to the very basic elements used by the framework and its plugins.
  • 5 jQuery 1.8.2 Library: The framework is currently dependant on jQuery version 1.8.2 as its base library. Also note the jQuery hashchange.js, this is required for IE7 as the hashchange event was not introduced until IE8.
    NOTE: Due to the use of '.live' the framework is not compatible with jQuery 1.9 for the moment.
  • 6 Modernizr: The framework uses Modernizr to optimise feature detection for modules and plugins.
  • 7 MandoMobile framework JavaScript: This file calls in all the necessary files and provides the initial loading interface for the framework. This is where all the magic happens!
  • 8 The 'data-view' attribute: This is the first mandatory attribute required for each view, this defines the area which contains the core view markup and states to the application which markup segment it should inject into the DOM.
    NOTE: This attribute has a value, this value is used by the application to identify the view's ID, without it the application navigation will not function correctly.
  • 9 The 'data-view-inner' attribute: A child '<div>' must be set inside the 'data-view' element named 'data-view-inner'. This is important for the application to calculate view heights correctly. It is essential that ALL of your markup sits as children inside this element.

So now you have the markup ready, lets take a look at getting around the application.
