Working with Headers and Footers

Using Static Headers and Footers

Static headers are static in the sense that they will remain in place at the head of the page whilst views transition in and out independently from them. This way header / footer content can persist and does not need to be repeated over and over.

Implementing a Static Header and Footer

Using this very simple set of markup, a simple static header and footer can be created:


<body>

1	<div data-header>
	
	</div>
	
	<div data-view="view_id">
				
		<div data-view-inner>
						
		</div>
		
	</div>
	
2	<div data-footer>
	
	</div>
	
<body/>
  • 1 The 'data-header' attribute: This attribute will define the '<div>' element as the header and everything inside it will remain during view transitions.
    NOTE: Only if your view has this element referenced on the landing / application start page will it appear in your application.
  • 2 The 'data-footer' attribute: This attribute will define the '<div>' element as the footer and everything inside it will remain during view transitions.
    NOTE: Only if your view has this element referenced on the landing / application start page will it appear in your application.

NOTE: A page can have a header, a footer, both or neither. It depends entirely on your application, the framework will handle measurements correctly taking into account the headers and footers into its view height calculations.
