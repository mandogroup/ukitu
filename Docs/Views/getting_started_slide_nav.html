Using the Slide Navigation Element

Why have Slide Navigation?

Navigation can be a problem on a mobile platform, page real estate is limited so a means to display the navigation without encroaching on the content has been created for you using the Slide Navigation element. The visibility of the navigation panel can be toggled using swipe gestures in addition to any reference you provide to an element to act as the activator.

Implementing Slide Navigation

Since this sits outside the view markup, similar to headers and footers this becomes a global element, look at the example below for correct placement. Any markup can be added to the element and with a little work the ScrollBox plugin can be used in conjunction with the Slide Navigation to help with a large navigation structure.
NOTE: The 'data-header' element is used here to demonstrate a global container for the activator, this is just and example it can be placed elsewhere if you wish.


<body>

1	<div data-slide-nav>
	
	</div>
	
	<div data-header>
	
2		<button data-slide-nav-activator>Navigation</button>
	
	</div>	
	
	<div data-view="view_id">
				
		<div data-view-inner>
						
		</div>
		
	</div>
	
<body/>
  • 1 The 'data-slide-nav' attribute: This attribute will define the '<div>' element as the Slide Navigation container. This element is hidden by default but can be accessed by using the activator (see below) which will toggle the Slide Navigation state on and off. As with the header and footer elements, this should sit outside of the 'data-view' element as it is a global feature, not specific to any given view.
    NOTE: Only if your view has this element referenced on the landing / application start page will it appear in your application.
