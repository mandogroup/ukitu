Working with Plugins

A small set of touch optimised plugins have been provided for you similar to those you might find in jQuery mobile. The key difference is that these plugins have been written with KnockoutJS; as this has become our preferred MVVM framework for applications, and touch in mind. This means that the UI elements should not have to be manually prompted to update and should react and respond as you would expect standard browser UI elements to.

Two categories of plugins are utilised in the framework; 'Core' Plugins and 'Optional' Plugins (which can be added to if you wish) but none are strictly framework dependent so feel free to use them on other non MandoMobile projects.

  • 'Core' Plugins: These are plugins which are automatically added to the MandoMobile a small subset which would be considered common enough to be always available, see the Plugins section for more details.
  • 'Optional' Plugins: These are plugins which are not included by default in the MandoMobile load stack. For your convinience the 'MandoMobile-config.js' file includes all of these that you can remove or comment out as necessary, just be aware that ommitting them from the options (discussed in the Advanced section) would eliminate the preload of that Plugin. See the Plugins section for more details.

Implementing Plugins

Plugins are implemented in one of two ways depending on your configuration setup as defined by the 'MandoMobile-config.js' file, the use of this file is outlined in the Advanced section but for now we will concentrate on the default config options and specifically toggling the automatic initilisation off, what that means and how to use it.

Lets have a look at a simple set of radio buttons and see how to implement a stylable radio toggle switch.

So this:


<div data-radio-toggle>
	
	<label for="radio1">Value 1</label>
	<input id="radio1" type="radio" name="group" value="1" />
	<label for="radio2">Value 2</label>
	<input id="radio2" type="radio" name="group" value="2" />
	<label for="radio3">Value 3</label>
	<input id="radio3" type="radio" name="group" value="3" />
	
</div>

Which Means...

By simply adding the correct markup, specifically focusing on the 'data-radio-toggle' wrapper in this instance, if the 'autoInitPlugins' option in 'MandoMobile-config.js' is set to true (which by default it is), the framework will attempt to build this control automatically upon page render.

A full reference with working examples can be found in the Plugins section which outlines each plugin and its usage, this is used a simple example.

Implementing a Plugin Manually

Whilst this method is quick and easy, in some more complicated applications you may wish to instantiate this plugin manually, this way you are able to retain local reference to the plugin, have more flexibility over options and be able to influence its behavior using JavaScript. The use for this is more common when dealing with presentational plugins as these can be more complex.

So with the 'autoInitPlugins' option set to false, the same markup is required but you would need to manually instantiate it in your JavaScript for this view. A simple function has been created for you to do this:
