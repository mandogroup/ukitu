Navigating Around

Navigating around your application project is easy, navigation can either be invoked by a simple addition to your '<a>' tags or via a simple call to the framework API in Javascript. The framework itself will take care of loading and managing the navigation structure complete with transitions.

NOTE: There is only currently only one page transition used in the framework, a slide of the existing page over to the left. New transitions will be added in time but do check the TODO list for details of intended future improvements.

Changing the Page with the <a> Tag


<a 1href="#view_id" 2data-url="/_docs/views/view_url.html" 3data-transition="fade">An Example Link</a>
  • 1 The 'href' attribute: The value must match the related page's 'data-view' value preceded by a hash. This allows the application to hook into the browser 'hashchange' event and use the value as the viewId to recall the view in the application.
  • 2 The 'data-url' attribute: This value is essential to define the actual location of your view on the server. This can be omitted when using the 'View Directory' feature. For more information see the Advanced section.
  • 3 The 'data-transition' attribute: This value is optional and to defines a the transition in which the page change should use. If this is ommitted the view change will use a standard sliding animation from right to left (see 'Transitions' below).

Changing the Page with Javascript


3mm.views.changeView(4'viewId', 5'viewUrl.html', 6'fade');
  • 3 The 'changeView' function call: This function sits within the 'views' namespace and can be called globally once the framework has been instantiated.
  • 4 The 'viewId' parameter: The value must match the related page's 'data-view' value preceded by a hash. This allows the application to hook into the browser 'hashchange' event and use the value as the viewId to recall the view in the application.
  • 5 The 'viewUrl' parameter: This value is essential to define the actual location of your view on the server. This can be omitted when using the 'View Directory' feature. For more information see the Advanced section.
  • 6 The 'viewTransition' parameter: Similar to the 'data-transition' attribute described above, this is an optional parameter which describes the transition you wish to use for that page change.

Transitions

Only two page transitions are currently available in the framework with more on the way in subsequent versions, these transitions are:

  1. 'swipe': Using the string identifier 'swipe' you are able to make the current view fade swipe left across the new view.
  2. 'fade': Using the string identifier 'fade' you are able to make the current view fade out above the new view.

NOTE: If no transition is specified, the default 'swipe' transition will be used.
